## Context

The project needs CI/CD quality gates to enforce code quality, security scanning, and test coverage on both PR and merge workflows. Since this is a macOS-specific project, tests are split into three categories with different run requirements.

## Decision

Three GitHub Actions workflows plus PR-Agent:

### pr-gate.yml
Runs on every PR (self-hosted macOS arm64 runner). Includes:
- `ruff format --check` (format)
- `ruff check` (lint)
- `pyright` (type check)
- `uv build` (build)
- `pytest --cov --cov-branch --cov-fail-under=90` (unit tests)
- `mutmut run --paths-to-mutate <changed>` (mutation tests on changed files)
- `vulture` (dead code detection)
- `trivy` filesystem scan (dependency audit)
- `semgrep` (security scanning)

### merge-gate.yml
Runs on push to main (self-hosted macOS arm64 runner). Includes everything in pr-gate plus:
- Full `mutmut run` (mutation testing, not just changed files)
- SonarCloud full analysis
- E2E tests (macOS runner only, if available)

### pr-agent.yml
Slash-command-only pattern (Pattern C):
- Triggers on `/review`, `/describe`, `/improve`, `/ask` comments
- Self-hosted macOS runner with `openai/glm-5.1` via `https://crof.ai/v1`
- Disabled by default via `ENABLE_PR_AGENT` repo variable
- Falls back when CodeRabbit is rate-limited

### SonarCloud
- Project key: `Jonathangadeaharder_mac-control-mcp`
- Organization: `jonathangadeaharder`
- Quality gate enforced on PRs via GitHub Actions
- Duplication exclusions configured as needed

### Branch Protection
- `main` branch requires `Required Checks (PR)` status check
- Requires 1 approving review (CodeRabbit or PR-Agent)
- Dismisses stale reviews

## Consequences

- Unit tests run on every PR (self-hosted macOS runner)
- Integration/E2E only tested on macOS
- CodeRabbit provides automated review; PR-Agent fallback via manual `/review` comment
- SonarCloud maintains quality history and enforces gates
- trivy + semgrep catch dependency vulnerabilities and security issues early
