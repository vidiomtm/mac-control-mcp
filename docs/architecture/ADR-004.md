## Context

The project combines macOS-native APIs (pyobjc, Quartz, Vision.framework) with pure Python logic. Tests must handle platform-specific code while maintaining high coverage.

## Decision

### Test Markers

Three pytest markers categorize tests:

| Marker | Scope | Requires macOS | Runs on CI |
|--------|-------|---------------|------------|
| `unit` | Pure logic — no system calls | No | Yes (ubuntu-latest) |
| `integration` | Subprocess/mdfind/AX calls | Yes | No (manual only) |
| `e2e` | Full JSON-RPC stdio roundtrip | Yes | No (manual only) |

### Coverage Requirements

- Branch coverage >=90% enforced via `pytest-cov`
- `pyproject.toml` config:
  ```toml
  [tool.pytest.ini_options]
  addopts = "-ra --strict-markers --cov --cov-branch --cov-fail-under=90 --cov-report=term-missing"
  ```
- Coverage XML uploaded to SonarCloud

### Live Test Control

Integration/E2E tests check `LIVE_TESTS` env var. Set `LIVE_TESTS=0` to skip live macOS calls. Default behavior auto-detects macOS availability.

### Mutation Testing

`mutmut` run on merge to main (not on every PR due to runtime). Verifies test suite quality by introducing code mutations and checking if tests catch them.

### Test File Layout

```
tests/
  __init__.py
  conftest.py           # Shared fixtures, mock setup, skip decorators
  unit/
    test_actions_keymap.py
    test_ax_fallback.py
    test_conftest.py
    test_kb.py
    test_kb_run_by_id.py
    test_osa_runner_unit.py
    test_security.py
    test_server_registration.py
    test_server_tools.py
    test_spotlight.py
    test_truncate.py
    test_vision.py
    test_vision_diff.py
  integration/
    test_kb_run.py
    test_osa_runner.py
    test_spotlight.py
    test_truncate_real_ax.py
  e2e/
    helpers.py
    test_kb_e2e.py
    test_stdio_roundtrip.py
```

### Mocking Strategy

- pyobjc frameworks mocked at import level in unit tests
- `subprocess.run` mocked for OSA runner tests
- `screencapture` binary mocked via `unittest.mock.patch`
- Integration tests use real macOS APIs with `LIVE_TESTS` guard
- E2E tests spawn real `uv run mac-control-mcp` process over stdio

## Consequences

- Unit tests run on any platform — fast CI feedback
- Integration/E2E only run on actual macOS hardware
- Coverage enforced regardless of test category
- Mutation testing ensures test quality, not just line coverage
- Clear separation between mock-isolated logic and real system interaction
