## Context

The Model Context Protocol (MCP) defines how AI agents interact with external tools. mac-control-mcp exposes macOS automation capabilities as MCP tools. The tool interface must balance expressiveness with security, token efficiency, and reliability.

## Decision

### Tool Categories

**1. AX (Accessibility) — 6 tools**

| Tool | Purpose | Returns |
|------|---------|---------|
| `ax_snapshot` | AX tree traversal for an app or system-wide | JSON with roles, labels, values, coords |
| `ax_click` | Mouse click at screen coords | Status + click params |
| `ax_type` | Type text into focused element | Status + length |
| `ax_scroll` | Scroll at screen position | Status + delta |
| `ax_hotkey` | Keyboard shortcut (e.g. `["cmd","c"]`) | Status + keys |
| `ax_system_ui` | System UI: Spotlight, menu bar, Control Center, Launchpad | Result or items |

**2. Vision — 4 tools**

| Tool | Purpose | Returns |
|------|---------|---------|
| `screen_capture` | Screenshot as base64 | Data, format, dimensions, size |
| `screen_list_windows` | List visible windows with IDs and bounds | Window list |
| `screen_ocr` | OCR from region or base64 image | Text + observations with confidence |
| `screen_wait_for_change` | Poll region until visual change detected | Change status + screenshot |

**3. OSA (AppleScript/JXA) — 3 tools**

| Tool | Purpose | Returns |
|------|---------|---------|
| `osa_search` | Fuzzy search the YAML KB for AppleScripts | Ranked entries with IDs |
| `osa_run` | Execute KB script by ID with args | Script output |
| `osa_exec` | Execute raw AppleScript/JXA (destructive patterns forbidden) | Script output |

**4. Apple Apps — 17 tools**

Mail (`mail_search`, `mail_recent`, `mail_send`), Calendar (`calendar_events`, `calendar_create_event`), Reminders (`reminders_list`, `reminders_add`), Notes (`notes_search`, `notes_get`, `notes_create`), Messages (`messages_recent`, `messages_send`), Contacts (`contacts_search`), Finder (`finder_tags_get`, `finder_tags_set`, `quicklook`), Spotlight (`spotlight_query`).

### Design Principles

1. **Coords-first interaction:** AX snapshots return screen coordinates → agents pass coords directly to `ax_click`/`ax_scroll`. Works with non-vision models.
2. **Token budget:** AX trees default to 12K chars. `prune_tree` + `trim_to_budget` control output size. Agents can override `budget_chars`.
3. **Security-first OSA:** `osa_exec` forbids destructive patterns. `osa_run` uses vetted KB entries. `check_osa_script()` validates at execution time.
4. **KB pattern over raw scripts:** Agents call `osa_search` → `osa_run` rather than writing AppleScript from scratch. YAML KB is the single source of truth for Apple app automation.
5. **Filtered at OS layer:** Calendar, Mail, Reminders queries accept date predicates that are pushed to the OS — no full data dumps.
6. **Fallback OCR:** Vision.framework OCR via pyobjc (fast) falls back to AppleScript-based OCR (compatible) when framework unavailable.

### Transport

- **Protocol:** JSON-RPC 2.0 over stdio
- **Entry point:** `mac-control-mcp` CLI via `__main__.py` → `server.run(transport="stdio")`
- **No HTTP endpoint** — server designed for local/sidecar execution

## Consequences

- 30 total tools covering full macOS UI automation surface
- KB pattern reduces LLM-generated script errors — search + execute known-good scripts
- Vision + AX provide complementary UI understanding (AX for native apps, Vision for Electron/games)
- Security guards prevent accidental destructive operations
- Token budgets prevent context window overflow from large AX trees
