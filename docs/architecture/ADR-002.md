# ADR-002: Toolchain

**Status:** Accepted
**Date:** 2026-05-17

## Context

The project requires a modern Python toolchain for package management, linting, formatting, type checking, testing, and mutation testing.

## Decision

Use the following toolchain:

| Concern | Tool | Version | Command |
|---------|------|---------|---------|
| Package manager | `uv` | latest | `uv add`, `uv sync`, `uv run` |
| Build backend | `hatchling` | via uv | configured in `pyproject.toml` |
| Linting | `ruff` | >=0.6 | `uvx ruff check` |
| Formatting | `ruff` | >=0.6 | `uvx ruff format` |
| Type checking | `pyright` | >=1.1.380 | `uvx pyright` |
| Testing | `pytest` | >=8.0 | `uv run pytest` |
| Coverage | `pytest-cov` | >=7.1 | integrated via pytest addopts |
| Mutation testing | `mutmut` | latest | `uv run mutmut run` |

Configuration lives in `pyproject.toml`:

- **ruff:** line-length 100, target py311, selects E/F/W/I/B/UP/SIM/RUF
- **pyright:** basic mode, checks `src/` only
- **pytest:** strict markers (unit/integration/e2e), asyncio_mode=auto, branch coverage >=90%
- **mutmut:** paths src + tests, runs via `uv run pytest --cov --cov-branch -x`

Python 3.11+ required. Dependencies managed via `uv.lock`.

## Consequences

- Single-command setup: `uv sync` installs everything
- Lint + format unified under `ruff` — no config duplication
- Branch coverage >=90% enforced as CI gate
- Mutation testing via `mutmut` verifies test suite quality
- pyright basic mode catches common type errors without being overly strict
