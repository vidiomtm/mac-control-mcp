## Context

mac-control-mcp is a unified macOS automation MCP (Model Context Protocol) server. It exposes macOS UI automation capabilities to AI agents via the MCP protocol over stdio transport.

The project needs to integrate four distinct macOS automation techniques under a single interface:

1. **AX (Accessibility) API** — via pyobjc `ApplicationServices` for accessibility tree snapshots and UI element interaction
2. **Vision/OCR** — via macOS `Vision.framework` for screen capture and text recognition
3. **OSA (AppleScript/JXA)** — via `osascript` subprocess for native Apple app automation
4. **Apple app integrations** — higher-level wrappers around OSA for Mail, Calendar, Reminders, Notes, Messages, Contacts, Finder, Spotlight

## Decision

Adopt a modular monorepo structure with domain-separated packages:

```
src/mac_control_mcp/
  __init__.py
  __main__.py        # CLI entry point
  server.py           # FastMCP server with all tool registrations
  security.py         # SSRF guard + forbidden OSA pattern detection
  truncate.py         # AX tree pruning for token budget
  ax/                 # Accessibility API layer
    snapshot.py       # AX tree traversal and serialization
    actions.py        # Mouse/keyboard/scroll CGEvent actions
    system_ui.py      # Spotlight, menu bar, Control Center, Launchpad
  vision/             # Screen capture and OCR layer
    capture.py        # Screenshot capture via screencapture CLI (optional PIL for scaling)
    ocr.py            # OCR via Vision.framework (direct + fallback)
    diff.py           # Screen region change detection
  osa/                # AppleScript/JXA layer
    kb.py             # YAML knowledge base with fuzzy search
    runner.py         # osascript subprocess execution
    knowledge/        # YAML script library for Apple apps
  apple/              # Apple-specific integrations
    spotlight.py      # mdfind predicate-based file search
```

Key architectural decisions:

- **Transport:** stdio only (JSON-RPC over stdin/stdout). No HTTP. Fits MCP client/server pattern.
- **Framework:** `mcp>=1.2` for FastMCP server. All tools are `@mcp.tool()` decorated functions.
- **Security:** `security.py` validates OSA scripts for destructive patterns (`rm -rf /`, `dd if=`, `sudo`) and blocks SSRF to private networks.
- **Token efficiency:** `truncate.py` prunes AX trees to fit token budgets (default 12K chars).
- **Knowledge base:** OSA scripts stored as YAML in `osa/knowledge/`, lazy-loaded, searchable via `rapidfuzz`.
- **No shell injection:** All subprocess calls use list-form args (`subprocess.run([...])`), never `shell=True`.

## Consequences

- Clear separation of concerns: AX, Vision, OSA each in own module
- Knowledge base pattern means most Apple app scripts never need Python changes — just YAML edits
- Vision.framework OCR available both directly (via pyobjc) and as AppleScript fallback
- AX actions use Quartz CGEvent API for reliable input simulation
- Tests separated by marker: unit (pure logic, run anywhere), integration (macOS + AX permissions), e2e (full stdio roundtrip)
